Stage 0: The Basics

How the Web Works

When a person goes to a web page like www.google.com, their computer sends a HTTP Request to a server.

The server finds the appropriate HTML document and sends it back to the user's computer where a web browser interprets the page and displays it on the user's screen. This video does a good job of explaining more if you can view it.

HTML

HTML, Hypertext Markup Language, documents form the majority of the content on the web.

HTML documents contain text content which describes "what you see" and CSS which describes "how it looks". This video gives a good overview if you can view it.

Tags and Elements

HTML documents are made of HTML elements. When writing HTML, we tell browsers the type of each element by using HTML tags. This video explains the distinction well if you can view it.

Computers do only what they are told because they interpret instructions literally. This makes them very unforgiving since a small mistake by a programmer can cause huge problems in a program.

Inline vs Block Elements

HTML elements are either inline or block. Block elements form an "invisible box" around the content inside of them.

Stage 3: Object Oriented Programming (OOP)

Basis

Object Oriented Programming (OOP) is designed by making them out of objects that interact with each other. They are class-based, meaning that objects are instances of classes. In Python, when calling a class function (like turtle), it calls the init function and it creates space in memory for the new object of the class (like Brad). This object can then access methods like forward and right in the Turtle class.

OOP is extremely helpful in programs especially large ones. For example, all tax rates were listed repeatedly in code in 27 files. So when a change is needed, a programmer would have to go through all the files and change the tax rates. Then all those changes would need to be tested and placed into production. If the tax rates are put in an OOP's tax class, this would be the only change needed. It is much simpler for the programmer and testers. It reduces repetition, which reduces operational risks (ie errors and not meeting the project's deadline) and is more efficient and effective.

This requires "Abstract thinking". This means finding similarity amongst different things. For example, in this nanodegree, it would be unfeasible for a programmer to exclusively program each of the hundred of web pages individually. This makes OOP needed to generate similar code to produce the HTML for different web pages and degree programs. Also in the HTML, we made all the titles and descriptions to have the same class names by type. So that in the CSS, we could use only one set of characteristics to control what each type (titles or descriptions) look like as a group. This saved unnecessary repetition and reduced the risk of errors.

Vocabulary:

Latency: is the time it takes a message to get from the source to the destination.connected.

Bandwidth: is the amount of information that can be transmitted per unit of time.connected.

Bit: is the smallest unit of information where 0=no and 1=yes.

Protocol: is a set of rules to tell how two can talk to each other.connected.

Server

Servers are computers running a server programs that responds to network requests. Servers will deliver either a static or a dynamic response depending on the content type requested. Static responses are pre-written files like images that are requested. Dynamic responses is built on-the-fly by programs running on the server.

HyperText Transfer Protocol (HTTP) is the most common protocol used on the world wide web. HTTP is how we (a client) communicate with a server. Our browser connects to the URL(Uniform Resource Locator). The URL example: http://www.google.com/index where http is the protocol, the host is google, and the path is /index. The two most important methods we learned are GET and POST.

GET: A GET is used for fetching documents. A client sends a GET request to a server and the server sends a response like a website to display. A GET request is a specific method of the HTTP protocol. Then it sends requests, asking the server to give responses. The request line example is |GET /path HTTP/1.1 |which is METHOD /path HTTP/version. The Server then reads this information and acts upon it. GET requests include the query parameters and values into the URL - therefore they also have a limited length of 2k characters. These parameters are in the URL. They are ok to cache and should not change servers.

POST: A POST requests alter data on the Server. POST requests on the other hand put them into the HTTP headers (after the headers) and parameters are in the body. There is no maximum limit but this can be configured. It is used for updating data. Unlike the GET, it is not ok to cache but it is ok to change servers.

Validation of Input

Validation is verification on the server side that only expected data is received. In the case of users submitting data via html forms, a defined set of input would always be expected. The server should not process just any data sent because a user can maliciously input large amounts of data to slow down or corrupt the server. This will prevent other users from using the website.

There are many ways to ensure the server receives only valid data like using a drop-down menu items or buttons would help but a malicious user can always mimic a form input and send data anyway. The most effective way is to verify all received data for compliance with expected data formats.

We used Jinja and it security features. It has an autoescape feature which is a secure procedure when building websites to avoid accidental oversight of security openings like entering HTML code in an Input box on a website.

HTML Templates and Abstraction

Templates are good because it keeps different languages separated. This also allows websites to have standard headers, footers and ect. Python is used as the main file that calls different HTML files that mesh into one website. This allows for standardization of a companies websites. Separate developers can work on different pieces. HTML code and CSS styles can be used minimally so if changes are needed only one file would need to be changed and not every webpage. This saves time, effort, and reduces the chance of mistakes!

We used Jinja and Google App Engine. Jinja has its own library which allows using Templates. It has its own programming logic for substituition, Loops, and If Then Elses.

Inheritance is also very useful. Jinja can reuse templates by inheritance of parent template elements to the child templates. The Jinja parent HTML template files and child templates go in the same folder. This gives us standardized headers or footers.

Abstraction is the hiding of detail of the Python Standard Library that we learned in Stage 3.

The benefits are to separate different types of code (Python from html), maintain code editor syntax highlighting, makes code more readable and secure websites. Html that is easier to modify. Template inheritance for html 'plug-in' functionality and consistency across pages. Repeated html elements with simple Jinja syntax and can use items like if-then-else with Jinja syntax.
